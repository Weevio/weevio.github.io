PunchOut enables enterprise customers to shop from their procurement systems (SAP Ariba, Coupa, etc.) and automatically see their negotiated pricing on your Shopify store - without logging in or leaving their procurement workflow.

## How It Works

<Step>
  <StepItem title="Procurement System Redirect">
    Enterprise buyer clicks "Shop Now" in their procurement system (SAP Ariba, Coupa, etc.). The system redirects to your Shopify store with a special `fromIdentity` parameter:

    ```
    https://your-store.com?fromIdentity=AN01234567890
    ```
  </StepItem>

  <StepItem title="Widget Captures Identity">
    The Weevio widget automatically:
    - Detects the `fromIdentity` parameter in the URL
    - Stores it in browser sessionStorage (12-hour expiration)
    - Resolves it to the matching Shopify customer via tag lookup
  </StepItem>

  <StepItem title="Customer Pricing Applied">
    The widget displays the customer's negotiated pricing throughout their browsing session:
    - Product pages show "Your Price"
    - Pricing persists across page navigation
    - No login required
  </StepItem>

  <StepItem title="Cart Return to Procurement">
    When ready to purchase, the cart data is sent back to the procurement system where the order is created using the negotiated pricing.
  </StepItem>
</Step>

## Setup Instructions

<Note type="warning">
**Prerequisite:** The fromIdentity tracker script must be installed on your Shopify theme for PunchOut to work. This script runs on all pages to capture the `fromIdentity` parameter when customers arrive from their procurement systems.
</Note>

### Step 1: Install Tracker Script in Shopify Theme

The tracker script must be added to your Shopify theme to capture the `fromIdentity` parameter on all pages.

<Step>
  <StepItem title="Navigate to Theme Editor">
    In your Shopify admin:
    1. Go to **Online Store** → **Themes**
    2. Find your active theme
    3. Click **Actions** → **Edit code**
  </StepItem>

  <StepItem title="Open theme.liquid File">
    In the left sidebar under **Layout**, click on **theme.liquid** to open the main theme file.
  </StepItem>

  <StepItem title="Add Tracker Script to Head Section">
    Find the `</head>` closing tag in the file (usually around line 100-200). Just **before** the `</head>` tag, add this script:

    ```liquid
    <!-- Weevio PunchOut Tracker -->
    <script src="https://cloud.weevio.co/ecommerce-widget/fromIdentity-tracker.js" defer></script>
    ```

    **Example placement:**
    ```liquid
    ...other head content...

    <!-- Weevio PunchOut Tracker -->
    <script src="https://cloud.weevio.co/ecommerce-widget/fromIdentity-tracker.js" defer></script>
  </head>
    ```
  </StepItem>

  <StepItem title="Save Changes">
    Click the **Save** button in the top right corner. The tracker script is now active on all pages of your store.
  </StepItem>
</Step>

<Note type="info">
**What does the tracker do?** This lightweight script (less than 1 KB) monitors the URL for the `fromIdentity` parameter on every page load and stores it in the browser's sessionStorage. This allows the pricing widget to identify the customer even as they navigate between pages.
</Note>

### Step 2: Understand FromIdentity Formats

The `fromIdentity` parameter is appended to your store URL by the procurement system. The Weevio widget recognizes these formats (case-insensitive):

**PunchOut formats:**
- `PunchOut Buyer:[customer_id]`
- `PunchOut_Buyer:[customer_id]`

**SAP Ariba formats:**
- `SAP Ariba Buyer:[customer_id]`
- `SAP_Ariba_Buyer:[customer_id]`
- `Ariba Buyer:[customer_id]`
- `Ariba_Buyer:[customer_id]`

**Character restrictions:**
- Maximum length: 50 characters
- Allowed characters: Alphanumeric, hyphens, underscores, colons, spaces
- Automatically sanitized for security

**Example URLs:**
```
https://your-store.com/products/laptop?fromIdentity=AN01234567890
https://your-store.com/collections/all?fromIdentity=AN01234567890
https://your-store.com?fromIdentity=AN01234567890
```

### Step 3: Tag Customers in Shopify

For PunchOut to work, you must tag each enterprise customer in Shopify with their `fromIdentity` value.

<Step>
  <StepItem title="Navigate to Customer in Shopify Admin">
    Go to **Customers** in your Shopify admin and open the customer record for the enterprise buyer.
  </StepItem>

  <StepItem title="Add FromIdentity Tag">
    Add a customer tag that matches the `fromIdentity` value their procurement system will send. Use one of the approved formats:

    **Recommended format:**
    ```
    Ariba Buyer: AN01631249352
    ```

    **Alternative formats:**
    ```
    SAP Ariba Buyer: AN01234567890
    PunchOut Buyer: AN01234567890
    ```

    <Note type="warning">
    The tag must **exactly match** the `fromIdentity` value sent by the procurement system (excluding prefix flexibility for spaces vs. underscores).
    </Note>
  </StepItem>

  <StepItem title="Assign Customer to Price List">
    Ensure the customer is assigned to the appropriate B2B price list in Shopify. This price list determines the negotiated pricing they'll see.
  </StepItem>
</Step>

**Example customer configuration:**

| Customer Name | Tag | Price List | Price Code Tag |
|:--------------|:----|:-----------|:---------------|
| ACME Manufacturing | `Ariba Buyer: AN01631249352` | Enterprise Tier (20% off) | `Price Code: ACME_20_OFF` |
| Regional Healthcare | `SAP Ariba Buyer: AN01631249352` | Contract Pricing | `Price Code: REGIONAL_HEALTH` |
| Tech Distributor - Division A | `PunchOut Buyer: AN01631249352` | Distributor Tier 1 | `Price Code: TECH_DIV_A` |

### Step 4: Test Your Configuration

<Step>
  <StepItem title="Create a Test URL">
    Build a test URL using your actual `fromIdentity` value:

    ```
    https://your-store.com?fromIdentity=AN01631249352
    ```

    You can test on any page - home, product, or collection pages.
  </StepItem>

  <StepItem title="Open Test URL in Browser">
    Navigate to your test URL. The widget should automatically capture the `fromIdentity` parameter.
  </StepItem>

  <StepItem title="Verify in Browser Console">
    Open browser developer tools (F12) and check the console:

    ```javascript
    // Check if fromIdentity was captured
    sessionStorage.getItem('weevio_fromIdentity')
    // Should return: "AN01631249352"

    // Check if it resolved to a customer
    sessionStorage.getItem('weevio_fromIdentity_customerId')
    // Should return: "gid://shopify/Customer/123456789"
    ```
  </StepItem>

  <StepItem title="Verify Pricing Display">
    Navigate to a product page. You should see:
    - "Your Price" displayed (if customer has negotiated pricing)
    - The correct discounted price for this customer
    - Pricing persists when navigating to other products
  </StepItem>
</Step>

**Common testing issues:**

- **FromIdentity not captured**: Check that the URL parameter is spelled correctly (`fromIdentity`, not `fromidentity`)
- **Customer not resolved**: Verify the customer tag in Shopify exactly matches the `fromIdentity` value
- **Pricing not showing**: Ensure the customer is assigned to a price list with active pricing rules
- **Session expired**: Sessions last 12 hours - clear sessionStorage or use a fresh browser tab to retest

## Cart Page Integration

The cart page integration extends PunchOut functionality to display B2B pricing throughout the shopping cart experience. This ensures enterprise buyers see their negotiated pricing from product pages all the way through checkout.

### Overview

When a customer arrives from their procurement system (via `fromIdentity`), the cart pricing integration:
1. Automatically detects B2B customer status
2. Fetches negotiated pricing for all cart items
3. Updates cart display with B2B prices
4. Populates `window.BSS_B2B.cart` for PunchOut Cloud integration

<Note type="info">
**Prerequisite:** The fromIdentity tracker script must already be installed (see Step 1 above). The cart pricing script builds on this foundation.
</Note>

### Installation

Add the cart pricing script to your Shopify cart template.

<Step>
  <StepItem title="Navigate to Cart Template">
    In your Shopify admin:
    1. Go to **Online Store** → **Themes**
    2. Click **Actions** → **Edit code**
    3. Under **Sections** or **Templates**, locate your cart template (usually `cart.liquid` or `main-cart.liquid`)
  </StepItem>

  <StepItem title="Add Configuration Script">
    Add this configuration **before** the cart pricing script. Place it in the `<head>` section or at the top of your cart template:

    ```liquid
    <script>
      window.weevioWidgetConfig = {
        shopifyDomain: '{{ shop.permanent_domain }}',
        teamID: 'YOUR_TEAM_ID',
        baseURL: 'https://api.weevio.com',
        cartBssPricing: {
          enableDomUpdates: true
        }
      };
    </script>
    ```

    Replace `YOUR_TEAM_ID` with your actual Weevio team ID.
  </StepItem>

  <StepItem title="Add Cart Pricing Script">
    Just after the configuration, add the cart pricing script:

    ```liquid
    <script src="https://cloud.weevio.co/ecommerce-widget/cartBssPricing.js" defer></script>
    ```

    **Complete example:**
    ```liquid
    <script>
      window.weevioWidgetConfig = {
        shopifyDomain: '{{ shop.permanent_domain }}',
        teamID: 'YOUR_TEAM_ID',
        baseURL: 'https://api.weevio.com',
        cartBssPricing: {
          enableDomUpdates: true
        }
      };
    </script>
    <script src="https://cloud.weevio.co/ecommerce-widget/cartBssPricing.js" defer></script>
    ```
  </StepItem>

  <StepItem title="Save Changes">
    Click **Save** to activate cart pricing for PunchOut customers.
  </StepItem>
</Step>

### Basic Configuration

The `cartBssPricing` object controls how B2B pricing is displayed in the cart:

```javascript
window.weevioWidgetConfig = {
  // ... other config ...
  cartBssPricing: {
    enableDomUpdates: true,  // Update cart UI with B2B prices
    debug: false              // Enable console logging for troubleshooting
  }
};
```

**Configuration Options:**

- **`enableDomUpdates`** (boolean, default: `true`)
  - `true`: Automatically update cart prices in the DOM
  - `false`: Data-only mode - populates `window.BSS_B2B.cart` without UI updates (useful if you're implementing custom pricing display)

- **`debug`** (boolean, default: `false`)
  - Enable detailed console logging for development and troubleshooting

### Theme-Specific Setup

#### Dawn Theme (and Most Standard Themes)

The default configuration works out of the box with Dawn and most standard Shopify themes:

```javascript
cartBssPricing: {
  enableDomUpdates: true
}
```

No additional selector configuration needed! The script automatically detects:
- Cart item rows
- Price elements
- Line totals
- Subtotal containers

#### Custom Themes

For custom or heavily modified themes, you may need to configure CSS selectors:

<Step>
  <StepItem title="Identify Cart Elements">
    Open your cart page in a browser and use Developer Tools (F12) to inspect:
    - Cart container element
    - Individual cart item rows
    - Unit price elements
    - Line total (item price × quantity) elements
    - Cart subtotal element
    - Variant ID attributes on cart items
  </StepItem>

  <StepItem title="Configure Custom Selectors">
    Add the `selectors` object to your configuration:

    ```javascript
    cartBssPricing: {
      enableDomUpdates: true,
      selectors: {
        cartContainer: '#my-custom-cart',
        cartItemRow: '.cart-item',
        cartItemPrice: '.item-unit-price',
        cartItemLinePrice: '.item-line-total',
        cartSubtotal: '#cart-subtotal-amount',
        variantIdAttribute: 'data-variant-id'
      }
    }
    ```
  </StepItem>

  <StepItem title="Test and Verify">
    Reload the cart page with a PunchOut customer session and verify prices update correctly. Enable `debug: true` to see detailed selector matching information in the console.
  </StepItem>
</Step>

**Custom Selector Reference:**

| Selector | Purpose | Example Value |
|:---------|:--------|:--------------|
| `cartContainer` | Main cart wrapper element | `'#my-cart'`, `'.cart-wrapper'` |
| `cartItemRow` | Individual cart line items | `'.cart-row'`, `'[data-cart-item]'` |
| `cartItemPrice` | Unit price per item | `'.price'`, `'.unit-price'` |
| `cartItemLinePrice` | Line total (price × qty) | `'.line-total'`, `'.item-total'` |
| `cartSubtotal` | Cart subtotal container | `'#subtotal'`, `'.cart-subtotal'` |
| `variantIdAttribute` | Attribute containing variant ID | `'data-variant-id'`, `'data-product-id'` |

#### BSS B2B App Users

If you're using the BSS: B2B Wholesale Solution app, the cart pricing script provides automatic compatibility:

```javascript
cartBssPricing: {
  enableDomUpdates: true
  // No special configuration needed!
}
```

The script automatically detects and works with BSS attributes:
- `bss-b2b-cart-item-key`
- `bss-b2b-variant-id`
- BSS-specific price containers

<Note type="success">
**Automatic Detection:** The script intelligently falls back to BSS selectors when detected, ensuring seamless integration with your existing BSS B2B setup.
</Note>

### Configuration Examples

#### Minimal Configuration (Dawn Theme)

Perfect for stores using Dawn or standard Shopify themes:

```javascript
window.weevioWidgetConfig = {
  shopifyDomain: '{{ shop.permanent_domain }}',
  teamID: 'YOUR_TEAM_ID',
  baseURL: 'https://api.weevio.com',
  cartBssPricing: {
    enableDomUpdates: true
  }
};
```

#### European Store (EUR with Custom Formatting)

Display prices in euros with European formatting conventions:

```javascript
cartBssPricing: {
  enableDomUpdates: true,
  formatting: {
    currencySymbol: '€',
    currencyPosition: 'after',     // "19,99 €" instead of "€19,99"
    decimalSeparator: ',',         // Use comma for decimals
    thousandsSeparator: '.',       // Use period for thousands
    decimalPlaces: 2
  }
}
```

**Result:** Prices display as `99.999,99 €` instead of `$99,999.99`

#### Show Original Price with Strikethrough

Display both original and B2B pricing to highlight savings:

```javascript
cartBssPricing: {
  enableDomUpdates: true,
  indicators: {
    showOriginalPrice: true,
    modifiedPriceClass: 'b2b-price',
    originalPriceClass: 'original-price'
  }
}
```

Add matching CSS to your theme:

```css
.original-price {
  text-decoration: line-through;
  opacity: 0.6;
  font-size: 0.9em;
}

.b2b-price {
  font-weight: bold;
  color: #007acc;
}
```

**Result:** Shows ~~$99.99~~ **$79.99** to emphasize the discount.

#### Custom Theme with Manual Selectors

For non-standard cart layouts:

```javascript
cartBssPricing: {
  enableDomUpdates: true,
  selectors: {
    cartContainer: '#custom-cart-container',
    cartItemRow: '.custom-cart-row',
    cartItemPrice: '.custom-unit-price span',

```

#### Data-Only Mode (No DOM Updates)

Use when implementing custom price display logic:

```javascript

```

Access pricing data programmatically:

### Price Formatting Options

Control how prices are formatted and displayed:

```javascript
cartBssPricing: {
  enableDomUpdates: true,
  formatting: {
    currencySymbol: '$',           // Currency symbol to display
    currencyPosition: 'before',    // 'before' ($19.99) or 'after' (19.99 €)
    decimalPlaces: 2,              // Number of decimal places (0-4)
    thousandsSeparator: ',',       // Thousands separator character
    decimalSeparator: '.'          // Decimal separator character
  }
}
```

**Formatting Examples:**

| Configuration | Result |
|:--------------|:-------|
| `{ currencySymbol: '$', currencyPosition: 'before' }` | `$1,234.56` |
| `{ currencySymbol: '€', currencyPosition: 'after' }` | `1,234.56 €` |
| `{ decimalSeparator: ',', thousandsSeparator: '.' }` | `$1.234,56` |
| `{ decimalPlaces: 0 }` | `$1,235` |
| `{ currencySymbol: 'USD', currencyPosition: 'after' }` | `1,234.56 USD` |

### Troubleshooting Cart Pricing

#### Prices Not Updating

**Symptoms:** Cart displays regular prices instead of B2B pricing

**Solutions:**
1. **Verify fromIdentity is captured**
   ```javascript
   // Check in browser console
   sessionStorage.getItem('weevio_fromIdentity')
   // Should return the customer's fromIdentity value
   ```

2. **Check selector configuration**
   - Enable debug mode: `debug: true`
   - Open browser console and look for selector matching errors
   - Verify selectors match your theme's HTML structure

3. **Verify customer setup**
   - Customer must be tagged with matching fromIdentity value
   - Customer must be assigned to a B2B price list
   - Price list must have active pricing rules

4. **Check script loading**
   ```javascript
   // Verify script loaded
   typeof window.weevioCartBssPricing !== 'undefined'
   // Should return true
   ```

#### Cart Item Keys Not Matching

**Symptoms:** Some cart items show B2B pricing, others don't

**Solutions:**
1. **Check variant ID attribute**
   - Use browser DevTools to inspect cart item elements
   - Verify the `variantIdAttribute` selector matches your theme
   - Common attributes: `data-variant-id`, `data-id`, `data-product-variant-id`

2. **Verify all variants have pricing**
   - Check that B2B price list includes all product variants
   - Some variants may not have negotiated pricing

#### Debug Mode for Development

Enable comprehensive logging during setup:

```javascript
cartBssPricing: {
  enableDomUpdates: true,
  debug: true
}
```

**Debug output includes:**
- Selector matching results
- Detected variant IDs
- API request/response details
- DOM update operations
- BSS B2B compatibility detection

**Console debugging commands:**

```javascript
// Check loaded configuration
console.log(window.weevioWidgetConfig.cartBssPricing);

// Verify B2B cart data
console.log(window.BSS_B2B?.cart);

// Check detected cart items
console.log(window.weevioCartBssPricing?.detectedItems);

// Force price refresh (if implemented)
window.weevioCartBssPricing?.refresh();
```

#### BSS B2B Compatibility Issues

**Symptoms:** Conflicts with BSS: B2B Wholesale Solution app

**Solutions:**
1. **Verify automatic detection**
   - The script should automatically detect BSS attributes
   - Check console for BSS detection messages (with `debug: true`)

2. **Disable DOM updates if needed**
   - Use data-only mode to avoid conflicts: `enableDomUpdates: false`
   - Implement custom display logic using `window.BSS_B2B.cart` data

3. **Contact support**
   - If issues persist, contact Weevio support with:
     - Theme name and version
     - BSS B2B app version
     - Browser console errors
     - Screenshot of cart HTML structure

## Technical Details

### Session Behavior

**Storage method:** Browser sessionStorage (per-tab isolation)

**Session lifetime:**
- Duration: 12 hours from initial capture
- Persistence: Survives page navigation within the same browser tab
- Expiration: Automatic after 12 hours or when browser tab is closed

**Multi-page persistence:**
Once `fromIdentity` is captured on the first page, it persists throughout the browsing session. Customers can navigate to any page without needing the `fromIdentity` parameter in every URL.

**Multiple tabs:**
Each browser tab maintains its own session. Different enterprise buyers can shop simultaneously in different tabs without conflict.

### Supported Procurement Systems

The Weevio widget supports any procurement system that can append a `fromIdentity` parameter to redirect URLs, including:

- **SAP Ariba** - Leading enterprise procurement platform
- **Coupa** - Cloud-based spend management
- **Oracle iProcurement** - Oracle's procurement solution
- **Jaggaer (SciQuest)** - Procurement and supply chain platform
- **Basware** - Purchase-to-pay automation
- **Ivalua** - Procurement and spend management
- **Custom procurement portals** - Any system supporting URL parameters

### Widget Configuration

No special widget configuration is required - `fromIdentity` detection is automatic and enabled by default.

## Troubleshooting

### Tracker Script Not Installed

**Symptoms:** Nothing happens when visiting a URL with `fromIdentity` parameter

**Solutions:**
- Verify the tracker script is added to theme.liquid in the `<head>` section
- Check that the script URL is correct: `https://cloud.weevio.co/ecommerce-widget/fromIdentity-tracker.js`
- Clear browser cache and reload the page
- Check browser console for script loading errors (404 errors)
- Ensure the theme changes were saved and published

### FromIdentity Not Detected

**Symptoms:** sessionStorage shows null for `weevio_fromIdentity`

**Solutions:**
- **First, verify tracker script is installed** (see above)
- Verify URL contains `?fromIdentity=` parameter (check spelling)
- Ensure JavaScript is not blocked by browser extensions
- Check that the parameter value is under 50 characters
- Verify allowed characters only (alphanumeric, hyphens, underscores, colons, spaces)

### Customer Not Resolved

**Symptoms:** sessionStorage shows fromIdentity but not customerId

**Solutions:**
- Verify customer exists in Shopify
- Check customer tag exactly matches the fromIdentity value
- Ensure tag uses an approved prefix format (e.g., "Ariba Buyer: ...")
- Verify tag has no extra spaces or typos

### Pricing Not Displayed

**Symptoms:** Customer resolved but no "Your Price" shown

**Solutions:**
- Confirm customer is assigned to a B2B price list
- Verify the price list has active pricing rules for the product
- Check browser console for API errors
- Ensure B2B pricing feature is enabled in Weevio settings

### Session Expired

**Symptoms:** Pricing worked earlier but not anymore

**Solutions:**
- Check if more than 12 hours have passed since initial visit
- Clear sessionStorage and revisit with `fromIdentity` parameter
- Open a fresh browser tab and start a new session
- Verify customer hasn't been untagged or removed from price list

For additional help, see the [Troubleshooting Guide](/docs/integrations/shopify/troubleshooting).
