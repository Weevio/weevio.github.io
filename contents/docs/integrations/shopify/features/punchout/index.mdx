The PunchOut Commerce feature enables seamless integration with enterprise procurement systems like SAP Ariba and Coupa, allowing business buyers to shop from their procurement platform without logging into your store while seeing their negotiated pricing.

[Screenshot: PunchOut session showing enterprise customer pricing]

## What It Does

PunchOut allows enterprise customers to:

- Click "Shop Now" in their procurement system
- Land on your Shopify store with a special identifier
- See their negotiated pricing automatically
- Shop without creating a Shopify account
- Cart items sync back to procurement system

## How It Works

### PunchOut Flow

```
┌─────────────────────────────────┐
│ 1. Procurement System           │
│    (SAP Ariba, Coupa)           │
│                                 │
│ User clicks "Shop at Vendor"    │
└────────────┬────────────────────┘
             │
             ▼
┌─────────────────────────────────┐
│ 2. Redirects to Shopify Store   │
│    with fromIdentity parameter  │
│                                 │
│ ?fromIdentity=AN01234567890     │
└────────────┬────────────────────┘
             │
             ▼
┌─────────────────────────────────┐
│ 3. Widget Captures FromIdentity │
│    - Stores in sessionStorage   │
│    - 12-hour expiration         │
│    - Resolves to customer ID    │
└────────────┬────────────────────┘
             │
             ▼
┌─────────────────────────────────┐
│ 4. Fetch Customer Pricing       │
│    - Call B2B API               │
│    - Get price list             │
│    - Display "Your Price"       │
└────────────┬────────────────────┘
             │
             ▼
┌─────────────────────────────────┐
│ 5. Customer Shops               │
│    - Browse products            │
│    - See negotiated prices      │
│    - Add items to cart          │
└────────────┬────────────────────┘
             │
             ▼
┌─────────────────────────────────┐
│ 6. Return to Procurement        │
│    - Cart data sent back        │
│    - Order created in system    │
│    - Purchase order generated   │
└─────────────────────────────────┘
```

## FromIdentity Formats

### Approved Prefixes

The widget recognizes these formats (case-insensitive):

**PunchOut:**
- `PunchOut Buyer:[customer_id]`
- `PunchOut_Buyer:[customer_id]`

**SAP Ariba:**
- `SAP Ariba Buyer:[customer_id]`
- `SAP_Ariba_Buyer:[customer_id]`
- `Ariba Buyer:[customer_id]`
- `Ariba_Buyer:[customer_id]`

### Example URLs

```
https://your-store.com/products/laptop?fromIdentity=AN01234567890

https://your-store.com/collections/all?fromIdentity=AN01234567890

https://your-store.com?fromIdentity=AN01234567890
```

### Character Restrictions

For security, fromIdentity values:

- **Maximum length:** 50 characters
- **Allowed characters:** Alphanumeric, hyphens, underscores, colons, spaces
- **Sanitized:** XSS protection applied
- **Case-insensitive:** Prefix matching is flexible

## Configuration

### Customer Setup

**Create customer mapping in Shopify:**

1. Go to Customers in Shopify Admin
2. Create or edit customer record
3. Add tag matching fromIdentity format
4. Assign to appropriate B2B price list

**Example customer tags:**
- `SAP Ariba Buyer:ACME_CORP_12345`
- `PunchOut Buyer:ENTERPRISE_ABC`
- `Ariba_Buyer:CUSTOMER_XYZ`

[Screenshot: Shopify customer with fromIdentity tag]

### Tag Matching

The widget matches exactly:

**URL parameter:**
```
?fromIdentity=AN01234567890
```

**Customer tag (must match):**
```
SAP Ariba Buyer:ACME_CORP_12345
```

**Note:** Matching is flexible on separator (space vs. underscore) but exact on ID.

### Widget Configuration

No special widget configuration needed! FromIdentity detection is automatic.

The widget:
- Detects fromIdentity in URL
- Stores in sessionStorage
- Resolves to customer ID
- Fetches pricing automatically

## Session Management

### Session Storage

FromIdentity is stored client-side:

```javascript
// Automatically stored by widget
sessionStorage.setItem('weevio_fromIdentity', 'SAP_Ariba_Buyer:ACME_CORP');
sessionStorage.setItem('weevio_fromIdentity_timestamp', Date.now());
sessionStorage.setItem('weevio_fromIdentity_customerId', 'gid://shopify/Customer/123');
```

### Session Lifetime

- **Duration:** 12 hours from initial capture
- **Storage:** Browser sessionStorage (cleared when browser closes)
- **Persistence:** Survives page navigation within session
- **Expiration:** Automatic after 12 hours

### Multi-Page Persistence

Once fromIdentity is captured:

1. Customer browses product pages
2. FromIdentity persists across pages
3. No need to include in every URL
4. Works for entire shopping session

## Troubleshooting

### FromIdentity Not Detected

**Check:**
- URL contains `?fromIdentity=` parameter
- Format matches approved prefixes
- No typos in parameter name
- JavaScript not blocked

**Debug:**
```javascript
// Check in browser console
console.log(sessionStorage.getItem('weevio_fromIdentity'));
```

### Customer Not Resolved

**Check:**
- Customer exists in Shopify
- Customer tag matches fromIdentity exactly
- Tag has correct prefix format
- Customer assigned to price list

### Pricing Not Showing

**Check:**
- Customer ID resolved correctly
- B2B API key configured
- Price list has pricing set
- No API errors in console

### Session Expired

**Check:**
- Less than 12 hours since first visit
- Browser not closed/restarted
- sessionStorage not manually cleared
- No privacy extensions blocking storage

See the [Troubleshooting Guide](/docs/integrations/shopify/troubleshooting) for more solutions.

console.log(sessionStorage.getItem('weevio_fromIdentity'));
