## Tag Format Overview

All visibility tags follow the naming pattern:

```
weevio.product.widget.cross_sell.<rule_type><optional_params>
```

Tags are added to **products** in Shopify admin and evaluated against **customer tags** when the cross-sell modal loads.

<Note type="warning">
**Case Sensitive:** All tags are case-sensitive. `weevio.product.widget.cross_sell.hide=true` and `Weevio.Product.Widget.Cross_Sell.Hide=True` are different tags.
</Note>

## Tag Types

### 1. Explicit Hide Tag

**Format:**
```
weevio.product.widget.cross_sell.hide=true
```

**Purpose:** Completely prevents product from appearing in cross-sell

**Evaluation Priority:** Highest (checked first, overrides all other rules)

**Parameters:**
- Must be exactly `=true` (no other values accepted)

**Use Cases:**
- Discontinued products still in catalog
- Internal-only items not for cross-sell
- Products being phased out
- Temporary removal during updates

**Example:**
```
Product: Old Model Widget
Tags:
- crossSell: true
- weevio.product.widget.cross_sell.hide=true

Result: Product NEVER appears in cross-sell for any customer
```

**Important:** This tag overrides show_if rules. Even if customer has matching show_if tags, product won't show if explicitly hidden.

---

### 2. Hide-If Customer Tag

**Format:**
```
weevio.product.widget.cross_sell.hide_if_customer_tag==<customer_tag_value>
```

**Purpose:** Hide product from customers who have specific tags

**Evaluation Priority:** Second (checked after explicit hide, before show_if)

**Parameters:**
- `<customer_tag_value>`: Exact customer tag to match (everything after `==`)
- Use `==` as delimiter (two equals signs)
- Multiple hide_if tags allowed per product

**Matching Logic:**
- Customer must have the EXACT tag (case-sensitive, character-for-character match)
- If customer has ANY hide_if tag → Product hidden
- Evaluation stops at first match (AND logic across multiple tags)

**Use Cases:**
- Hide warranty products from customers with existing coverage
- Hide retail products from wholesale accounts
- Hide budget options from premium customers
- Hide competitive products from specific resellers

**Example 1: Single Hide-If Tag**
```
Product: AppleCare Extended Warranty
Tags:
- crossSell: true
- weevio.product.widget.cross_sell.hide_if_customer_tag==weevio.customer.ace.eligible=true

Customer A Tags: ["weevio.customer.ace.eligible=true"] → Product HIDDEN
Customer B Tags: ["retail-customer"] → Product SHOWN (default)
Customer C Tags: [] → Product SHOWN (default)
```

**Example 2: Multiple Hide-If Tags (OR Logic)**
```
Product: Consumer Grade Product
Tags:
- crossSell: true
- weevio.product.widget.cross_sell.hide_if_customer_tag==wholesale
- weevio.product.widget.cross_sell.hide_if_customer_tag==b2b-account
- weevio.product.widget.cross_sell.hide_if_customer_tag==enterprise

Customer A Tags: ["wholesale"] → Product HIDDEN
Customer B Tags: ["b2b-account"] → Product HIDDEN
Customer C Tags: ["enterprise"] → Product HIDDEN
Customer D Tags: ["retail"] → Product SHOWN
Customer E Tags: ["wholesale", "premium"] → Product HIDDEN (first match wins)
```

**Tag Format Requirements:**
- Must use exactly `==` (not `=`, not `===`)
- Everything after `==` is the customer tag to match
- No spaces around `==` (unless space is part of the tag value)
- Tag value can contain spaces, special characters, dots, etc.

---

### 3. Show-If Customer Tag

**Format:**
```
weevio.product.widget.cross_sell.show_if_customer_tag==<customer_tag_value>
```

**Purpose:** Product ONLY shows to customers with specific tags

**Evaluation Priority:** Third (checked last, after hide rules)

**Parameters:**
- `<customer_tag_value>`: Exact customer tag to match (everything after `==`)
- Use `==` as delimiter (two equals signs)
- Multiple show_if tags allowed per product

**Matching Logic:**
- If NO show_if tags exist → Product shows by default (unless hidden)
- If show_if tags exist:
  - Customer has at least ONE matching tag → Product shown
  - Customer has NO matching tags → Product hidden
- OR logic across multiple show_if tags (any match wins)

**Use Cases:**
- Show premium products only to VIP customers
- Show enterprise features only to qualified accounts
- Show seasonal promotions to opted-in customers
- Show tier-specific products to membership levels

**Example 1: Single Show-If Tag**
```
Product: VIP Exclusive Accessory
Tags:
- crossSell: true
- weevio.product.widget.cross_sell.show_if_customer_tag==vip

Customer A Tags: ["vip"] → Product SHOWN
Customer B Tags: ["regular"] → Product HIDDEN
Customer C Tags: [] → Product HIDDEN
Customer D Tags: ["vip", "premium"] → Product SHOWN
```

**Example 2: Multiple Show-If Tags (OR Logic)**
```
Product: Premium Tier Product
Tags:
- crossSell: true
- weevio.product.widget.cross_sell.show_if_customer_tag==weevio.customer.ace.tier=2
- weevio.product.widget.cross_sell.show_if_customer_tag==weevio.customer.ace.tier=3
- weevio.product.widget.cross_sell.show_if_customer_tag==ACE Tier 2
- weevio.product.widget.cross_sell.show_if_customer_tag==ACE Tier 3

Customer A Tags: ["weevio.customer.ace.tier=2"] → Product SHOWN
Customer B Tags: ["ACE Tier 3"] → Product SHOWN
Customer C Tags: ["weevio.customer.ace.tier=1"] → Product HIDDEN
Customer D Tags: ["regular-customer"] → Product HIDDEN
Customer E Tags: ["weevio.customer.ace.tier=2", "ACE Tier 3"] → Product SHOWN (any match)
```

**Important Behavior:**
- **Default changes:** If NO show_if tags exist, product shows by default. Once you add show_if tags, default becomes HIDE unless customer matches.
- **Opt-in model:** Show-if creates an opt-in system where only tagged customers see the product.

## Evaluation Flow

The widget evaluates visibility in this exact order:

<Step>
  <StepItem title="Step 1: Check Explicit Hide">
    **Check:** Does product have `weevio.product.widget.cross_sell.hide=true` tag?

    **If YES:** Skip product entirely (don't show in cross-sell)

    **If NO:** Continue to Step 2
  </StepItem>

  <StepItem title="Step 2: Evaluate Hide-If Rules">
    **Check:** Does product have `hide_if_customer_tag` tags?

    **If NO hide_if tags:** Continue to Step 3

    **If YES:**
    - Loop through all hide_if tags
    - Check if customer has ANY matching tag
    - **If customer has match:** Hide product (stop evaluation)
    - **If no matches:** Continue to Step 3
  </StepItem>

  <StepItem title="Step 3: Evaluate Show-If Rules">
    **Check:** Does product have `show_if_customer_tag` tags?

    **If NO show_if tags:** Show product (default behavior)

    **If YES:**
    - Loop through all show_if tags
    - Check if customer has at least ONE matching tag
    - **If customer has match:** Show product
    - **If no matches:** Hide product
  </StepItem>
</Step>

## Conflict Resolution

### Hide Wins Over Show

If a customer tag matches BOTH a hide_if rule AND a show_if rule, the hide rule takes priority.

**Example:**
```
Product Tags:
- weevio.product.widget.cross_sell.hide_if_customer_tag==wholesale
- weevio.product.widget.cross_sell.show_if_customer_tag==wholesale

Customer Tags: ["wholesale"]

Evaluation:
1. No explicit hide → Continue
2. Customer has "wholesale" matching hide_if → HIDE
3. Show_if evaluation never reached

Result: Product HIDDEN
```

**Why:** Hide rules are considered safety/exclusion rules and should be respected even if show rules exist.

### Multiple Tags Use OR Logic

**For Hide-If (Any Match Hides):**
```
Product has: hide_if_customer_tag==A, hide_if_customer_tag==B

Customer with ["A"] → Hidden
Customer with ["B"] → Hidden
Customer with ["A", "B"] → Hidden
Customer with ["C"] → Shown
```

**For Show-If (Any Match Shows):**
```
Product has: show_if_customer_tag==A, show_if_customer_tag==B

Customer with ["A"] → Shown
Customer with ["B"] → Shown
Customer with ["A", "B"] → Shown
Customer with ["C"] → Hidden
```

### No Tags = Default Behavior

```
Product Tags:
- crossSell: true
(no visibility tags)

Result: Product shows to ALL customers (default cross-sell behavior)
```

## Tag Validation Rules

### Format Requirements

**Valid:**
```
weevio.product.widget.cross_sell.hide=true
weevio.product.widget.cross_sell.hide_if_customer_tag==vip
weevio.product.widget.cross_sell.show_if_customer_tag==premium
```

**Invalid:**
```
weevio.product.widget.cross_sell.hide=false  ❌ (must be =true or omit tag)
weevio.product.widget.cross_sell.hide_if_customer_tag=vip  ❌ (must use ==)
weevio.product.widget.cross_sell.show_if_customer_tag===vip  ❌ (must use exactly ==)
Weevio.Product.Widget.Cross_Sell.hide=true  ❌ (wrong capitalization)
```

### Customer Tag Matching

**Matching is EXACT:**
- Case-sensitive
- Character-for-character comparison
- No partial matching
- No wildcards

**Examples:**
```
Product tag: hide_if_customer_tag==VIP
Customer tag: "vip"  ❌ (case mismatch)
Customer tag: "VIP"  ✓ (exact match)

Product tag: show_if_customer_tag==weevio.customer.tier=1
Customer tag: "weevio.customer.tier=1"  ✓ (exact match)
Customer tag: "weevio.customer.tier=2"  ❌ (different value)

Product tag: hide_if_customer_tag==wholesale account
Customer tag: "wholesale"  ❌ (partial match doesn't work)
Customer tag: "wholesale account"  ✓ (exact match with space)
```

### Recommended Limits

- **Maximum tags per product:** 10 visibility tags (performance)
- **Maximum show_if tags:** 5 (maintainability)
- **Maximum hide_if tags:** 5 (maintainability)
- **Customer tag length:** Under 50 characters (readability)

<Note type="warning">
**No Server-Side Validation:** Invalid tag formats are silently ignored. Products with malformed tags will show default behavior (visible to all customers unless other rules apply).
</Note>

## Tag Combination Patterns

### Pattern 1: Explicit Hide (Simplest)

```
Use: Product should never show
Tags: hide=true
Result: No one sees product
```

### Pattern 2: Hide from Specific Group

```
Use: Hide from one customer segment
Tags: hide_if_customer_tag==excluded-group
Result: All customers see it EXCEPT those with "excluded-group" tag
```

### Pattern 3: Show to Specific Group

```
Use: Only show to one customer segment
Tags: show_if_customer_tag==included-group
Result: ONLY customers with "included-group" see it
```

### Pattern 4: Show to Multiple Groups (OR Logic)

```
Use: Show to several customer segments
Tags:
- show_if_customer_tag==group-a
- show_if_customer_tag==group-b
- show_if_customer_tag==group-c
Result: Customers with ANY of these tags see it
```

### Pattern 5: Show to Group BUT Hide Exceptions

```
Use: Target group but exclude specific members
Tags:
- show_if_customer_tag==vip
- hide_if_customer_tag==vip-suspended
Result:
- "vip" only → Shown
- "vip-suspended" → Hidden (hide wins)
- "vip" + "vip-suspended" → Hidden (hide wins)
```

### Pattern 6: Complex Multi-Segment

```
Use: Multiple target groups with multiple exclusions
Tags:
- show_if_customer_tag==tier-2
- show_if_customer_tag==tier-3
- show_if_customer_tag==enterprise
- hide_if_customer_tag==restricted
- hide_if_customer_tag==probation
Result:
- Must have tier-2, tier-3, OR enterprise
- Even with above, hidden if restricted OR probation
```

## Implementation Notes

### Server-Side Evaluation

All tag evaluation happens server-side in the cross-sell API for:
- **Security:** Customers can't manipulate visibility rules client-side
- **Performance:** No client-side tag parsing overhead
- **Consistency:** Same rules applied everywhere

### Tag Storage

- **Product tags:** Stored in Shopify product metadata
- **Customer tags:** Stored in Shopify customer metadata
- **Sync:** Tags sync to Weevio Cloud during product/customer API calls
- **Cache:** Tag evaluation results cached per session

### Performance Impact

- Tag evaluation adds less than 5ms to cross-sell API response
- Cached after first evaluation per session
- No impact on page load time (async)

## Advanced Topics

### Tag Inheritance

**Products do NOT inherit tags** from collections, variants, or parent products. Each product must have its own visibility tags.

### Guest Customers

Guest customers (not logged in) have **no tags** and see:
- Products with NO show_if tags (default visible)
- Products hidden from guests if show_if tags exist

### B2B vs B2C

Use customer tags to differentiate:
```
B2B customers: "b2b-account"
B2C customers: "retail-customer"

Show B2B products:
show_if_customer_tag==b2b-account

Hide B2B products from retail:
hide_if_customer_tag==retail-customer
```
